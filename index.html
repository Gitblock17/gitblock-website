Integration Examples

import requests

response = requests.post(
    "https://api.gitblock.io/v1/chat/completions",
    headers={
        "Authorization": "Bearer YOUR_API_KEY",
        "Content-Type": "application/json"
    },
    json={
        "model": "llama-3.3-70b",
        "messages": [{"role": "user", "content": "Hello!"}]
    }
)
print(response.json()["choices"][0]["message"]["content"])
from openai import OpenAI

client = OpenAI(
    api_key="YOUR_API_KEY",
    base_url="https://api.gitblock.io/v1"
)

response = client.chat.completions.create(
    model="llama-3.3-70b",
    messages=[{"role": "user", "content": "Hello!"}]
)
print(response.choices[0].message.content)
from langchain_openai import ChatOpenAI

llm = ChatOpenAI(
    api_key="YOUR_API_KEY",
    base_url="https://api.gitblock.io/v1",
    model="llama-3.3-70b"
)

result = llm.invoke("Hello!")
print(result.content)
curl -X POST https://api.gitblock.io/v1/chat/completions \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "llama-3.3-70b",
    "messages": [{"role": "user", "content": "Hello!"}]
  }'
const response = await fetch("https://api.gitblock.io/v1/chat/completions", {
    method: "POST",
    headers: {
        "Authorization": "Bearer YOUR_API_KEY",
        "Content-Type": "application/json"
    },
    body: JSON.stringify({
        model: "llama-3.3-70b",
        messages: [{ role: "user", content: "Hello!" }]
    })
});
const data = await response.json();
console.log(data.choices[0].message.content);

D

Start Building

Send chat completions, embeddings, and more. Full OpenAI API compatibility.

Code Examples

Use It Like Any API

Drop-in compatible with the OpenAI SDK. Switch one line and you're live.

Chat Completion 
from openai import OpenAI

client = OpenAI(
    api_key="gb_xxxx",
    base_url="https://api.gitblock.io/v1"
)

resp = client.chat.completions.create(
    model="llama-3.3-70b",
    messages=[{"role": "user", "content": "Explain quantum computing in 3 sentences."}]
)
print(resp.choices[0].message.content)
Streaming 
stream = client.chat.completions.create(
    model="llama-3.3-70b",
    messages=[{"role": "user", "content": "Write a haiku."}],
    stream=True
)

for chunk in stream:
    if chunk.choices[0].delta.content:
        print(chunk.choices[0].delta.content, end="")
Code Generation 
resp = client.chat.completions.create(
    model="deepseek-coder-33b",
    messages=[{
        "role": "user",
        "content": "Write a Python function to find prime numbers."
    }]
)
code = resp.choices[0].message.content
print(code)

Run a Node 
# Run a contributor node with Docker
docker run -d \
  --name gitblock-node \
  -e NODE_KEY=your_node_key \
  -e MODELS=llama-3.3-70b \
  -p 8080:8080 \
  gitblock/node:latest

# Check node status
curl http://localhost:8080/health

FAQ

Frequently Asked Questions

Everything you need to know about GitBlock.

There's no catch. GitBlock is sustained by a network of contributor nodes who donate spare compute. Think of it like SETI@home but for AI inference. Node operators earn reputation and governance tokens, not dollars.
All traffic is end-to-end encrypted. Processing nodes receive encrypted payloads and return encrypted responses. No prompts are logged, stored, or used for training. The node operator never sees plaintext.
Your wallet address is used to deterministically generate your API key. This means no sign-up forms, no email collection, and no central database of users. Your key is derived from your address and can be regenerated anytime.
Currently 89 models are available including Llama 3.3, DeepSeek Coder, Qwen 2.5, Mistral, CodeLlama, Phi-3, Gemma 4, and Yi 1.5. New models are added as contributors bring them online.
Yes! Running a node is as simple as pulling the Docker image and configuring your models. You'll need a GPU with at least 8GB VRAM for smaller models. See our GitHub for detailed setup instructions.
GitBlock uses the exact same API format as OpenAI. The only difference is the base URL. Performance is comparable for most tasks, with sub-200ms latency. The key difference: it's completely free and decentralized.
There are soft limits to prevent abuse (currently 100 requests per minute per key). If you need higher throughput, you can run your own node or request a limit increase via GitHub issues.
